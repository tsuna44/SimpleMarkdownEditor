# SimpleMarkdownEditor 要求書

## 1. 概要

SimpleMarkdownEditor は、Markdown テキストをリアルタイムにプレビューしながら編集できるデスクトップアプリケーションである。
Python / PyQt6 で実装され、macOS および Windows で動作する。

---

## 2. 動作環境

| 項目 | 要件 |
|---|---|
| OS | macOS 12 以降 / Windows 10 以降 |
| Python | 3.10 以降 |
| 依存ライブラリ | PyQt6 ≥ 6.6、PyQt6-WebEngine ≥ 6.6、markdown ≥ 3.5、pygments ≥ 2.17 |
| ネットワーク | Mermaid.js を CDN から取得するため、初回表示時にインターネット接続が必要 |

---

## 3. 機能要件

### 3.1 エディタ

| ID | 要件 |
|---|---|
| ED-01 | プレーンテキストとして Markdown を入力できること |
| ED-02 | 行番号ガターを左端に常時表示すること |
| ED-03 | 水平スクロールを有効にし、行の折り返しを行わないこと |
| ED-04 | タブ幅はスペース 2 文字分とすること |
| ED-05 | カーソル位置（行・列）をステータスバーにリアルタイム表示すること |
| ED-06 | 文字数・単語数をステータスバーにリアルタイム表示すること |

### 3.2 リアルタイムプレビュー

| ID | 要件 |
|---|---|
| PV-01 | エディタの内容変更から 300 ms 後にプレビューを自動更新すること（デバウンス処理） |
| PV-02 | プレビューはエディタと左右に並べて表示すること（スプリッター分割） |
| PV-03 | スプリッターをドラッグしてエディタ・プレビューの幅比率を変更できること |

### 3.3 Markdown レンダリング

| ID | 要件 | 構文例 |
|---|---|---|
| MD-01 | 見出し（H1〜H3）を表示すること | `# H1` `## H2` `### H3` |
| MD-02 | 太字・斜体・打ち消し線をレンダリングすること | `**太字**` `*斜体*` `~~打消~~` |
| MD-03 | インラインコードをハイライト表示すること | `` `code` `` |
| MD-04 | フェンスドコードブロックに Pygments によるシンタックスハイライトを適用すること | ```` ```python ``` ```` |
| MD-05 | テーブルをレンダリングすること | `\| 列1 \| 列2 \|` |
| MD-06 | 引用ブロックをレンダリングすること | `> 引用` |
| MD-07 | 箇条書き・番号付きリストをレンダリングすること | `- 項目` `1. 項目` |
| MD-08 | 水平線をレンダリングすること | `---` |
| MD-09 | リンクをレンダリングすること | `[text](url)` |
| MD-10 | 画像を最大幅 100% で表示すること | `![alt](url)` |

### 3.4 Mermaid ダイアグラム

| ID | 要件 |
|---|---|
| ME-01 | ```` ```mermaid ```` フェンスで囲まれたブロックを Mermaid ダイアグラムとしてレンダリングすること |
| ME-02 | ダイアグラムにホバーすると「Copy PNG」「SVG ダウンロード」ボタンを表示すること |
| ME-03 | Copy PNG ボタンで PNG 画像をクリップボードにコピーできること（2× Retina 解像度） |
| ME-04 | SVG ボタンでダイアグラムを SVG ファイルとして保存できること |
| ME-05 | フォントサイズ変更に連動してダイアグラムのスケールが即時更新されること |

### 3.5 ツールバー（編集補助）

| ID | 要件 |
|---|---|
| TB-01 | 見出し挿入ボタン（H1 / H2 / H3）を提供すること |
| TB-02 | 太字 (`Ctrl+B`)・斜体 (`Ctrl+I`) のトグル挿入ボタンを提供すること |
| TB-03 | 打ち消し線・インラインコードの挿入ボタンを提供すること |
| TB-04 | 箇条書き・番号付きリスト・引用・コードブロックの挿入ボタンを提供すること |
| TB-05 | Mermaid テンプレート・テーブルテンプレートの挿入ボタンを提供すること |
| TB-06 | リンク挿入ボタン (`Ctrl+K`) を提供すること |
| TB-07 | 水平線の挿入ボタンを提供すること |
| TB-08 | テキストを選択した状態で挿入ボタンを押すと、選択範囲を装飾で囲むこと |

### 3.6 検索・置換

| ID | 要件 |
|---|---|
| SR-01 | `Ctrl+F` で検索・置換ダイアログを開けること |
| SR-02 | モードレスダイアログとして表示し、編集を妨げないこと |
| SR-03 | 前へ（↑）・次へ（↓）でマッチ間を移動できること |
| SR-04 | 現在位置のマッチを置換する「置換」機能を提供すること |
| SR-05 | すべてのマッチを一括置換する「すべて置換」機能を提供すること |
| SR-06 | 大文字・小文字を区別するオプション（Aa）を提供すること |
| SR-07 | 正規表現検索オプション（.*）を提供すること |
| SR-08 | マッチ件数と現在位置（例: `3/12`）を表示すること |

### 3.7 ファイル操作

| ID | 要件 | ショートカット |
|---|---|---|
| FI-01 | 新規ファイルを作成できること | `Ctrl+N` |
| FI-02 | Markdown ファイル（`.md`, `.markdown`）およびテキストファイルを開けること | `Ctrl+O` |
| FI-03 | 現在のファイルを上書き保存できること | `Ctrl+S` |
| FI-04 | 別名保存（Save As）ができること | — |
| FI-05 | A4 縦向きでプレビュー内容を PDF にエクスポートできること | `Ctrl+Shift+E` |
| FI-06 | 未保存の変更がある状態でファイルを閉じる・切り替える場合、保存確認ダイアログを表示すること | — |
| FI-07 | 未保存の変更をウィンドウタイトルおよびファイル名ラベルの「●」で示すこと | — |
| FI-08 | コマンドライン引数でファイルパスを指定して起動できること | — |

### 3.8 テーマ

| ID | 要件 |
|---|---|
| TH-01 | ライトテーマとダークテーマを切り替えられること |
| TH-02 | テーマ切り替えはツールバーのボタン（☀ / 🌙）で行えること |
| TH-03 | テーマ切り替え時にエディタ・プレビュー・UI クロームすべてが即時更新されること |
| TH-04 | プレビューのシンタックスハイライトテーマもテーマに連動すること（`github` / `github-dark`） |

### 3.9 フォントサイズ

| ID | 要件 |
|---|---|
| FS-01 | ツールバーの「A−」「A+」ボタンでフォントサイズを変更できること |
| FS-02 | フォントサイズの範囲は 8px〜32px とすること |
| FS-03 | エディタとプレビューのフォントサイズが連動して変更されること |
| FS-04 | 現在のフォントサイズをツールバーに数値で表示すること |

---

## 4. 非機能要件

| ID | 要件 |
|---|---|
| NF-01 | エディタへの入力からプレビュー更新までのレイテンシは 300 ms 以内（デバウンス後）とすること |
| NF-02 | ライト・ダーク両テーマで、UI 要素のコントラスト比が十分に確保されていること |
| NF-03 | macOS・Windows の両プラットフォームで同一のソースコードが動作すること |
| NF-04 | CDN 未接続環境でも、Mermaid 以外の機能（編集・プレビュー・ファイル操作）は正常に動作すること |

---

## 5. ビルド要件

| ID | 要件 |
|---|---|
| BL-01 | `build.sh` を実行することで、macOS 向け `.app` バンドルを生成できること |
| BL-02 | `build.sh` を Git Bash（Windows）で実行することで、Windows 向け `.exe` を生成できること |
| BL-03 | `build_windows.bat` を実行することで、CMD / PowerShell からも Windows 向け `.exe` を生成できること |
| BL-04 | `--onefile` オプションで単一実行ファイルとしてビルドできること |
| BL-05 | `--clean` オプションでビルド成果物（`build/`・`dist/`）を削除してからビルドできること |
| BL-06 | PyInstaller が未インストールの場合、ビルドスクリプトが自動でインストールすること |

---

## 6. ソースファイル構成

```
Python/
├── markdown_editor.py   エントリポイント（main 関数）
├── main_window.py       メインウィンドウ（MarkdownEditor クラス）
├── editor_widget.py     行番号付きエディタウィジェット（CodeEditor クラス）
├── search_dialog.py     検索・置換ダイアログ（SearchDialog クラス）
├── preview_html.py      プレビュー用シェル HTML 生成
├── themes.py            ライト / ダークテーマ定数
├── build.sh             ビルドスクリプト（macOS / Windows Git Bash 共通）
├── build_windows.bat    ビルドスクリプト（Windows CMD / PowerShell 用）
└── requirements.txt     依存パッケージ一覧
```

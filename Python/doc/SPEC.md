# Simple Markdown Editor — Python 版 仕様書

## 概要

Python と PySide6 (Qt6) で実装されたデスクトップ向け Markdown エディタ。
タブ形式で複数ファイルを同時に開くことができ、左ペインでテキスト編集、右ペインでリアルタイムプレビューを表示する分割レイアウト。

---

## 動作環境

| 項目 | 要件 |
|---|---|
| Python | 3.10 以上 |
| 主要依存ライブラリ | PySide6 >=6.10, <6.12、markdown >= 3.5、pygments >= 2.17 |
| Java | 17 以上（PlantUML 機能を使う場合のみ） |
| ライセンス | MIT（依存ライブラリは LGPL/BSD） |

インストール:

```bash
pip install -r requirements.txt
```

起動:

```bash
python markdown_editor.py [file.md]
```

## 機能仕様

### 1. エディタ

#### 1-1. テキスト入力

- プレーンテキスト編集（`QPlainTextEdit` ベース）
- 等幅フォント優先使用: JetBrains Mono → Consolas → Menlo → Courier New
- 折り返しなし（水平スクロール対応）
- タブ幅: スペース 2 文字分

#### 1-2. 行番号ガター

- エディタ左端に行番号を常時表示
- スクロールに追従してリアルタイム更新
- テーマに応じた配色（`surface2` / `gutter` カラー）

#### 1-3. カーソル位置表示

- ステータスバーに `Ln {行}, Col {列}` をリアルタイム表示

#### 1-4. 外部ファイル変更の自動リロード

- `QFileSystemWatcher` で開いているファイルを監視
- 外部エディタ等で変更・保存された場合、カーソル位置を維持しつつ自動でコンテンツを更新
- アプリ自身の保存中は誤検知しない（`_saving` フラグで制御）

---

### 2. Markdown レンダリング

#### 2-1. リアルタイムプレビュー

- テキスト変更から 300 ms のデバウンス後に自動更新（PlantUML 未キャッシュ時は 800 ms）
- Python 側の `markdown` ライブラリで HTML 変換し、WebEngine に注入

#### 2-2. サポートする Markdown 構文

| 構文 | 詳細 |
|---|---|
| 見出し | `#` `##` `###` (h1〜h6) |
| 太字 | `**text**` |
| 斜体 | `*text*` |
| 取り消し線 | `~~text~~`（Python 側で前処理して `<del>` 変換） |
| インラインコード | `` `code` `` |
| コードブロック | ```` ```lang ``` ```` (フェンス形式) |
| テーブル | GFM 形式 |
| 引用 | `> text` |
| 箇条書き / 番号付きリスト | `- item` / `1. item` |
| 水平線 | `---` |
| リンク | `[text](url)` |
| 画像 | `![alt](url)` |
| 脚注 | `[^1]` 形式（markdown.extensions.extra） |
| `<details>` 内 Markdown | `markdown="1"` を自動付与（md_in_html 拡張） |
| アンカーリンク | TocExtension により見出しにアンカーを付与（`slugify_unicode` 使用） |
| Mermaid ダイアグラム | ```` ```mermaid ```` ブロック（ローカル vendor JS でレンダリング） |
| PlantUML ダイアグラム | ```` ```plantuml ```` ブロック（ローカル Java で SVG 生成） |

#### 2-3. シンタックスハイライト

- Pygments によるコードブロックのハイライト
- ライトテーマ: `github` スタイル
- ダークテーマ: `github-dark` スタイル
- CSS クラス: `.codehilite`

#### 2-4. Mermaid ダイアグラム

- ```` ```mermaid ```` ブロックを検出し `<div class="mermaid" data-src="...">` に変換
- `vendor/mermaid.min.js` をローカルで読み込みレンダリング（elkjs レイアウトエンジン対応）
- ライト / ダークテーマに対応
- フォントサイズ変更時に SVG を viewBox ベースでスケール再計算

**ダイアグラムアクションボタン**（ホバーで表示）:

| ボタン | 動作 |
|---|---|
| 📋 Copy PNG | SVG を 2× 解像度（Retina 対応）の PNG に変換しクリップボードにコピー |
| ⬇ SVG | SVG ファイルとしてダウンロード（保存ダイアログ表示） |

#### 2-5. PlantUML ダイアグラム

- ```` ```plantuml ```` ブロックを検出し、ローカルの PlantUML を使って SVG に変換
- `@startuml` / `@enduml` ヘッダが省略されている場合は自動補完
- レンダリング結果を最大 50 件キャッシュ（FIFO で古いものから破棄）

**PlantUML コマンド自動検出（優先順位順）:**

| 優先度 | 検出パス |
|---|---|
| 1 | `plantuml` (PATH) |
| 2 | `plantuml.sh` (PATH) |
| 3 | `vendor/plantuml.jar`（同梱 jar） |
| 4 | Homebrew OpenJDK + 同梱 jar |
| 5 | システム java + Homebrew/apt/標準パスの plantuml.jar |

PlantUML が見つからない場合はエラーメッセージをプレビューに表示。

**PlantUML 設定ダイアログ**（File > Settings）:

| 設定項目 | デフォルト | 選択肢 |
|---|---|---|
| JVM ヒープメモリ (-Xmx) | `1g` | 512m / 1g / 2g / 4g |
| 図サイズ上限 (px) | `32768` | 4096〜65536 |

#### 2-6. 同時スクロール

- エディタのスクロール位置（比率）をプレビュー側に自動反映（エディタ → プレビュー）
- コンテンツ更新後 150 ms のタイマーで再同期

### 3. ファイル操作

| 操作 | ショートカット | 詳細 |
|---|---|---|
| 新規タブ | Ctrl+T | 空の untitled.md タブを追加 |
| 開く | Ctrl+O | 複数選択可。空タブがあれば再利用、なければ新規タブで開く |
| 上書き保存 | Ctrl+S | 未保存時は「名前を付けて保存」にフォールバック |
| 名前を付けて保存 | — | フィルタ: `.md` `.markdown` `.txt` |
| タブを閉じる | Ctrl+W | 未保存確認あり。最後の 1 タブは閉じず空タブに切り替え |
| PDF エクスポート | Ctrl+Shift+E | A4 / 縦向き / 余白 15mm、`printToPdf()` を使用 |
| 印刷 | Ctrl+P | システムプリンタへ `QPrintDialog` 経由で印刷 |
| Drag & Drop | — | ファイルをウィンドウにドロップして開く（複数可） |

**未保存確認ダイアログ**:
新規作成・ファイルを開く・タブを閉じる・ウィンドウを閉じる際に、未保存の変更がある場合は「保存 / 破棄 / キャンセル」を提示。

**最近使ったファイル**:
- File メニュー内「最近使ったファイル」サブメニューに最大 10 件を表示
- 設定は `settings.ini` に JSON 形式で永続化
- 「履歴をクリア」で全件削除

**ウィンドウタイトル**:
`Markdown Editor — {ファイル名}` / 未保存変更がある場合はファイル名末尾に ` ●` を付加。

**リンクナビゲーション**:
- http/https リンク → システムブラウザで開く
- file:// `.md` リンク → エディタで開く（既存タブがあれば再利用）
- アンカーリンク（同一 HTML ファイル内）→ スクロールで遷移

---

### 4. タブ管理

| 操作 | 方法 |
|---|---|
| 新規タブ | Ctrl+T またはタブバー右端の `+` ボタン |
| タブを閉じる | Ctrl+W またはタブ右端の `×` ボタン |
| タブの並び替え | ドラッグ＆ドロップ |
| タブ切り替え | クリック |

- タブが 0 になると自動的に空の untitled.md タブを生成（常に 1 タブ以上を維持）
- ファイルを開く際、現在のタブが空かつ未変更の場合は新規タブを作らず再利用

---

### 5. テーマ

#### 5-1. ライトテーマ / ダークテーマ

ツールバーの ☀ / 🌙 ボタンで切り替え（再起動不要）。

| 要素 | ライト | ダーク |
|---|---|---|
| 背景 | `#f5f4f0` | `#161512` |
| テキスト | `#1a1814` | `#e8e4df` |
| アクセント | `#c0603a` | `#e07a52` |
| サーフェス | `#ffffff` | `#1e1c1a` |

テーマ切り替え時の更新対象:
- エディタの配色（`QPalette` 経由）
- ツールバー・ステータスバー・ダイアログ（QSS 一括適用）
- プレビュー HTML（テーマ別シェル HTML を再生成してリロード）
- Pygments CSS（ダークは `github-dark`、ライトは `github`）

---

### 6. フォントサイズ変更

ツールバーの `A−` / `A+` で変更（範囲: 8〜32 px）。

| 対象 | 動作 |
|---|---|
| エディタ | `QFont.setPointSize()` で即時反映 |
| プレビュー本文 | `document.body.style.fontSize` を JavaScript で設定 |
| Mermaid SVG | `scaleSvg()` で viewBox × スケール係数を再計算してリサイズ |

現在のサイズはツールバー中央に `{n}px` として表示。

---

### 7. 検索・置換ダイアログ

ショートカット: `Ctrl+F`（モードレスダイアログ、開くと検索フィールドを自動選択）

| 機能 | 詳細 |
|---|---|
| テキスト検索 | 入力と同時にリアルタイム検索・ハイライト |
| 前へ / 次へ | `↑` / `↓` ボタン、件数表示 `{n}/{total}` |
| 大文字小文字区別 | `Aa` チェックボックス |
| 正規表現 | `.*` チェックボックス（`QRegularExpression` 使用） |
| 1件置換 | 現在選択中の一致を置換 |
| 全件置換 | すべての一致を一括置換、完了後に件数表示 |

---

### 8. 編集ツールバー（クイック挿入）

| ボタン | 挿入内容 | ショートカット |
|---|---|---|
| H1 / H2 / H3 | `# ` / `## ` / `### ` を行頭に挿入 | — |
| B | `**選択テキスト**` | Ctrl+B |
| I | `*選択テキスト*` | Ctrl+I |
| ~~ | `~~選択テキスト~~` | — |
| ` | `` `選択テキスト` `` | — |
| • List | `- ` を行頭に挿入 | — |
| 1. List | `1. ` を行頭に挿入 | — |
| > Quote | `> ` を行頭に挿入 | — |
| ``` Code | ```` ```\ncode\n``` ```` | — |
| Mermaid | Mermaid サンプルブロック挿入 | — |
| PlantUML | PlantUML サンプルブロック挿入 | — |
| Table | 3列テーブルのテンプレート挿入 | — |
| ≡T | カーソル位置のテーブルを整形 | Ctrl+Shift+T |
| Link | `[テキスト](url)` | Ctrl+K |
| --- | `\n---\n` | — |

テキスト選択中は選択範囲をラップ、未選択時はプレースホルダーを挿入してカーソルをプレースホルダー上に配置。

#### テーブル整形（≡T）

カーソルがMarkdownテーブル内にある状態で実行すると、列幅を揃えて再整形する。

- セパレーター行（`|---|`）のアライメント指定（`:---:` 等）を保持
- 列数が不均一な場合は空セルで補完

---

### 9. アウトラインパネル

ショートカット: `Ctrl+Shift+O`（左側ドックとして表示/非表示トグル）

- H1〜H3 見出しを抽出してリスト表示
- クリックでエディタのカーソルを該当行に移動
- エディタのテキスト変更から 300 ms 後に自動更新

---

### 10. ファイルツリーパネル

ショートカット: `Ctrl+Shift+F`（左側ドックとして表示/非表示トグル）

- 現在編集中ファイルのディレクトリをルートとして表示
- `.md` / `.markdown` / `.txt` ファイルのみ表示（その他は非表示）
- ダブルクリックでファイルを開く

---

### 11. ステータスバー

| 表示項目 | 内容 |
|---|---|
| `Ln {n}, Col {n}` | 現在のカーソル行・列番号 |
| `{n} words` | 単語数（スペース区切り） |
| `{n} chars` | 総文字数 |
| バージョンバッジ | `mermaid vX \| elkjs vX \| plantuml vX`（`vendor/VERSIONS.json` から取得） |
| `● Standalone` | 実行モードバッジ（右端固定） |

## 既知の制限・注意事項

| 項目 | 内容 |
|---|---|
| Mermaid バージョン | `vendor/mermaid.min.js` を差し替えることでバージョン変更可 |
| PlantUML の依存 | Java と PlantUML が未インストールの場合、PlantUML ブロックはエラー表示になる |
| XSS | Markdown の変換結果を `innerHTML` に直接挿入しており、DOMPurify 等のサニタイズは未実装 |
| PDF 出力 | Mermaid / PlantUML 図のレンダリング完了前に PDF 生成すると図が欠けることがある |
| LGPL + PyInstaller | `--onefile` ビルドは PySide6 の静的バンドルとみなされる可能性あり。`--onedir` を使用すること |
